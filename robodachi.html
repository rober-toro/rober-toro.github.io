ROBODACHI

Planet of Robots Project

A step-driven mobile RPG concept that turns real-world walking into exploration, resource collection, character growth, and marketplace interaction.

Duration2022-2024
ScopeMobile UX · GameFi · IP System

01. Project Snapshot

A mobile RPG driven by real-world movement.

Planet of Robots is a mobile RPG game product that turns real-world walking into exploration, resource collection, character growth, and marketplace interaction.

ROBODACHI is the core character IP within the Planet of Robots universe. Players' real-world movement drives their ROBODACHI companion to automatically explore the game world.

My Role

I independently led the project from early concept to pre-production design, including product strategy, UX system architecture, core gameplay logic, mobile interaction flows, prototype design, user incentive structure, economy design, visual direction, character IP system, and supporting 2D/3D assets.

Project Coverage

Mobile Game UX · Product System Design · GameFi Experience · Character IP System

02. Design Challenge

Supporting casual motivation and Web3 value without unnecessary complexity.

Many walking reward products end at "Walk → Reward → End." Rewards become an external outcome rather than part of the core experience, which creates low long-term engagement.

Many GameFi products focus heavily on token earning but lack meaningful gameplay, emotional attachment, and reasons to return beyond financial rewards. They also create a high onboarding barrier for casual users.

How might we turn walking into a long-term engaging experience instead of a one-time reward mechanic?

Movement should support exploration, resource collection, upgrades, bonding, gear, world expansion, and a scalable character IP system.

03. Target Users & Motivation Strategy

Layered motivations without increasing casual-user complexity.

Different users enter the product for different reasons, so the UX needed to support fun, progression, low-friction participation, ownership, and marketplace value without forcing every player into the same path.

Traditional & Casual Players

Need fun, progression, challenge, character attachment, simple rewards, and low-friction participation.

Blockchain & Investment Users

Need token utility, NFT assets, ownership, marketplace connection, and external value exchange.

04. Product Solution

A step-driven RPG progression loop.

I designed a progression loop where real-world movement becomes the input for in-game exploration, resource collection, equipment discovery, and character growth. Walking fuels the larger game system, while progression motivates users to keep walking.

Walk → Explore → Collect Resources → Upgrade ROBODACHI → Unlock New Actions → Walk Again

Core Action Loop

Move → Gather → Process → Build → Upgrade → Repair → Trade → Expand

05. Product System Architecture

Movement, gameplay, and economy layers.

After defining the core progression loop, I mapped how exploration, resource collection, refinement, character upgrades, land expansion, and ownership mechanics connect across the experience. This clarified the relationship between player-facing actions and the deeper economy layer.

Movement LayerReal-world walking and distance tracking.
Game LayerGrowth, exploration, town building, materials, bonding, and gear.
Economy LayerMarketplace, token flow, NFT assets, item exchange, and external ownership.
Design GoalKeep deeper systems understandable from the player's core actions.

06. Mobile Prototype Evolution

From flow validation to a refined player experience.

The prototype moved from early functional screens toward a fuller product system and a more polished visual direction.

Version 1 - Flow Validation

The first version defined the repeatable sequence: select ROBODACHI, start exploration, collect rewards, and return to the map. It tested navigation clarity and key gameplay states before adding advanced systems.

Version 2 - System Expansion

The second prototype tested a fuller mobile product across character management, modular customization, hub navigation, gear inspection, selling actions, and bonding status.

Version 3 - Refined Visual Direction

The third version refined the presentation quality and brought the concept closer to the intended player experience.

07. Character System & IP Design

A scalable system of five ROBODACHI unit types.

ROBODACHI was designed as a scalable character system with five unit types. Each type had a different personality, movement logic, potential gameplay function, and collection role.

  • Flying Unit - aerial scouting, high mobility, and high energy cost
  • Humanoid Unit - flexibility and balanced actions
  • Runner Unit - speed, ground scouting, and exploration
  • Lifter Unit - strength, protection, and construction
  • Cube Unit - intelligence, processing, and strategy

08. Attributes & Build Logic

Visual identities connected to gameplay roles.

To make ROBODACHI scalable as a character system, I explored different build variations. Each build could support gameplay directions such as mobility, stamina, strength, processing, gathering, or exploration efficiency.

09. Gear & Equipment Progression

Customization with gameplay value.

Gear was designed as both a visual customization layer and a gameplay modifier. Different items could change how each ROBODACHI performed in exploration, battle, resource gathering, or town maintenance.

Gear supports personalization, performance changes, long-term collecting, upgrading, repair, and marketplace logic.

10. World & Biome Expansion

An explorable isometric world tied to step-driven progression.

To support long-term progression, I explored multiple biome variations. Each biome could introduce new materials, environmental themes, challenges, and collectible opportunities.

Real-world walking allowed ROBODACHI to explore different areas, discover resources, and unlock new routes over time.

11. 3D Model Development & Concept Video

Extending the visual direction into product storytelling.

The visual direction was designed to feel futuristic, friendly, modular, and collectible. ROBODACHI was planned as an expandable IP ecosystem across digital collectibles, social campaigns, 3D animation, physical toys, community events, and interactive content.

I developed the ROBODACHI 3D model and storyboard, then collaborated with an external animation team to translate these assets into a concept video that communicated the product world, core use cases, and brand vision.

12. Product Website

Communicating the world, character system, and launch bridge.

The website concept introduced the ROBODACHI universe, explained model types and attributes, presented the prototype direction, and used campaign and marketplace-like sections to support early community engagement.

13. Early Engagement: Lightweight HTML Game

Immediate interactive value before the full mobile product.

Because the full mobile game required a longer development cycle, the NFT release needed an earlier interactive experience that could provide immediate value to holders.

I designed a lightweight HTML game as an interim playable experience. It was not intended to represent the final gameplay of Planet of Robots; instead, it served as an early engagement layer that allowed NFT holders and community members to interact with the ROBODACHI IP before the full product was ready.

A launch bridge that kept early holders engaged and gave the community something playable before full development.

14. IP Ecosystem Extension

Extending ROBODACHI beyond the mobile game.

I planned ROBODACHI as a scalable character IP ecosystem including community collaborations, physical mystery box toys, streetwear, online events, and interactive comics.

I also explored a physical mystery box toy concept using a magnetic attachment system, allowing randomized parts such as top accessories, arms, body shells, and legs to be swapped and recombined. This connected the in-game customization system with a collectible physical product strategy.
