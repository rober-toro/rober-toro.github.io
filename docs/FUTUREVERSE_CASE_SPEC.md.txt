## 3. Challenge

Headline:

**Making Web3 onboarding understandable and trustworthy for Japanese users.**

Body:

Futureverse users needed to understand FuturePass, wallet connection, sign-in methods, Root Rewards, referrals, token-related actions, and ecosystem participation. The challenge was not only translation, but reducing friction, building trust, and creating repeatable communication for Japanese users.

Challenge cards:

1. **Complex wallet and sign-in flows**
   Users had to understand email login, MetaMask, wallet connection, verification, and FuturePass ownership.

2. **Web3 terminology barrier**
   Concepts like token swaps, rewards, staking, referrals, and wallet signatures needed clearer Japanese explanations.

3. **Community education across channels**
   LINE, webinars, guides, and campaign assets needed consistent localized messaging.

4. **Feedback back to the design team**
   User questions, analytics, interviews, and Google Forms helped identify friction points and communication gaps.

## 7. Onboarding Flow System

Create two large cards.

### A. Email-Based FuturePass Onboarding

User type:
Web3 beginners

Goal:
Reduce wallet-first friction.

Flow:
Open FuturePass → choose email → enter email → verify code → complete account

UX note:
This path made FuturePass feel more approachable for users who were not ready to begin with a crypto wallet.

### B. MetaMask-Based FuturePass Onboarding

User type:
Web3-ready users

Goal:
Support wallet ownership and ecosystem access.

Flow:
Open MetaMask browser → access FuturePass → choose MetaMask → approve connection/signature → complete account

UX note:
This path supported users who already understood wallet-based access while still requiring clear step-by-step Japanese guidance.

Visuals:
Use screenshots or guide images from the raw folder if available. If screenshots are too small, use them as large image cards with captions.

## 8. Education Content Library

Create a grid of 3–4 cards.

Cards:

### FuturePass Account Creation

User goal:
Create a FuturePass account through email or MetaMask.

Communication issue:
Users needed to understand the difference between simple email onboarding and wallet-based onboarding.

### Root Rewards Registration & Redeem

User goal:
Register for Root Rewards, understand points, referrals, and redeem flow.

Communication issue:
Users needed to understand why they should return after account creation.

### CENNZ → ROOT Swap Guide

User goal:
Understand how to swap or migrate tokens through FuturePass-related flows.

Communication issue:
This was more advanced and required clear prerequisite explanation.

### Sign-Up Method Comparison

User goal:
Choose the right sign-in or wallet connection method.

Communication issue:
Too many login options can create decision friction for new users.
